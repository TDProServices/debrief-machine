- **Flexible timing**: bridge can happen any evening, not exactly at specific time
- **Permission to be messy**: notebook capture never needs to be perfect
- **No judgment system**: failed transfers or missed days are learning data, not failures
- **Energy-based adjustments**: scale system intensity based on available executive function

### **System Sustainability Protection:**
- **Regular simplification reviews**: monthly check for unnecessary complexity
- **Value-based filtering**: keep only elements that generate clear business or personal value
- **Integration protection**: ensure system enhances rather than competes with existing workflows
- **Evolution permission**: allow system to change based on what actually works
