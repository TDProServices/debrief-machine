## üé¨ **SESSION-SPECIFIC GUIDELINES**

### **Executive Summary Format**
- **Session purpose** in first bullet (why did this session happen?)
- **Key achievements** in 2-3 bullets (what was accomplished?)
- **Breakthrough moments** if any (what changed our understanding?)
- **Session impact** on overall timeline (how does this advance the project?)

### **Timeline Development Focus**
- **Session as timeline unit** - each session = discrete timeline entry
- **Clear progression markers** - how does this session build on previous ones?
- **Decision points documentation** - what choices were made and why?
- **Momentum tracking** - is the project accelerating, slowing, or pivoting?

### **Cross-Session Continuity**
- **Handoff notes** for next session (what should be tackled next?)
- **Open loops tracking** (what remains unresolved?)
- **Context preservation** (what background is essential for next session?)
- **Strategic question development** (what should drive the next session?)

## üîß **TECHNICAL IMPLEMENTATION STANDARDS**

### **Artifact Creation Requirements**
- **Single artifact per session** - avoid fragmentation
- **Downloadable format** - always markdown for portability
- **Version control ready** - include metadata for tracking
- **Platform-agnostic** - works regardless of AI platform used

### **Documentation Quality Metrics**
1. **Scannability Test** - Can key points be identified in 30 seconds?
2. **Action Clarity Test** - Are next steps immediately obvious?
3. **Context Completeness Test** - Can someone understand this session without reading previous ones?
4. **Continuity Test** - Does this enable seamless continuation in next session?
